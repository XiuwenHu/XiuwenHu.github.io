音视频

date: 2016-07-02 20:27:05

audio 音频

标签定义声音，比如音乐或其他音频流。

属性：

autoplay ：音频在就绪后马上自动播放。

controls ：向用户显示控件，比如播放按钮。

loop ：每当音频结束时重新开始播放。

preload ：音频在页面加载时进行加载，并预备播放。
如果使用 “autoplay”，则忽略该属性。

src ：要播放的音频的 URL。

source 媒介

< source > 标签为媒介元素（比如 < video > 和 < audio >）定义媒介资源。
< source > 标签允许您规定可替换的视频/音频文件供浏览器根据它对媒体类型或者编解码器的支持进行选择。

track 字幕

< track > 标签为诸如 video 元素之类的媒介规定外部文本轨道。
用于规定字幕文件或其他包含文本的文件，当媒介播放时，这些文件是可见的。

default :规定该轨道是默认的，假如没有选择任何轨道。

kind=”captions|chapters|descriptions|metadata|subtitles”
表示轨道属于什么文本类型。

label :轨道的标签或标题。

src :轨道的 URL。

srclang=”language_code” :轨道的语言，若 kind 属性值是 “subtitles”，则该属性必需的。

video 视频

定义视频，比如电影片段或其他视频流。

属性：

autoplay：视频在就绪后马上自动播放。

controls：向用户显示控件，比如播放按钮。

height ：设置视频播放器的高度。

width :设置视频播放器的宽度。

loop :当媒介文件完成播放后再次开始播放。

preload :视频在页面加载时进行加载，并预备播放。
如果使用 “autoplay”，则忽略该属性。

src :要播放的视频的 URL。

embed 嵌入

定义嵌入的内容，比如插件。

属性：

type ：定义嵌入内容的类型。

src :要播放的视频的 URL。

css3选择器

这里说一下css3选择器的一些类型和作用，主要是说明一下除了一般常用的#ID，.class，标签选择器等选择器。

用E和F代表父级和子级,则有下列情况：

子元素选择器&后代选择器

两者分别可写为：

E > F

子元素选择器，选中的是子元素F，但子孙不会被选中



E F

后代选择器，是选择所有的F，并包括子子孙孙

相邻兄弟选择器 & 通用兄弟选择器

两者分别可写为：

E ＋ F

相邻兄弟选择器 ：1.必需是同级的 2.必须是相邻的 3.必需是在下方的 4、有且只有一个



E ~ F

通用兄弟选择器 ：1.必需是同级的 2.必需是在下方的 3.可以选择多个

属性选择器

E[attr]

E[attr=“value"]

E[attr~=“value”] ~：表示几个以空格隔开的属性值中包含了“value”

E[attr^=“value”] ^:以value开头的值

E[attr$=“value”] $:以value结尾的值

E[attr*=“value"] *：包含了value的值

E[attr|=“value”] |: 以value或value-开头

其中E[attr="value"]和E[attr*="value"]是最实用的

伪类选择器

input:focus{}

input:disabled{}

input:checked{}

input:enabled{}

:nth 选择器

:first-child 选择某个元素的第一个子元素

:last-child 选择某个元素的最后一个子元素

:nth-last-child()选择某个元素的一个或多个特定的子元素，从这个元素的最后一个子元素开始算；

:nth-of-type()选择指定的元素；

:nth-last-of-type()选择指定的元素，从元素的最后一个开始计算；

:first-of-type选择一个上级元素下的第一个同类子元素；

:last-of-type选择一个上级元素的最后一个同类子元素；

:only-child选择的元素是它的父元素的唯一一个子元素；

:only-of-type选择一个元素是它的上级元素的唯一一个相同类型的子元素；

:empty选择的元素里面没有任何内容。

:nth-child()选择某个元素的一个或多个特定的子元素；

:nth-child(length);/参数是具体数字/

:nth-child(n);/参数是n,n从1开始计算/

:nth-child(2n); /n的倍数选择，n从1开始算/

:nth-child(n+length); /选择大于length后面的元素/

:nth-child(-n+length); /选择小于length前面的元素/

:nth-child(2n+1); /表示隔几选一/ odd even

###否定选择器（:not）
否定选择器:not()，可以定位不匹配该选择器的元素。

比如，如果需要定义表单元素中的input元素，但是又不想包括submit类型的input的时候时可以这样使用：

input:not([type=”submit”]) {border: 1px solid red;}

伪元素

::first-letter 选择文本块的第一个字母，除非在同一行里面包含一些其它元素，不过这个主要运用于段落排版上。

::first-line 选择元素的第一行

::before,::after 这两个主要用来给元素的前面或后面插入内容，这两个常用”content”配合使用，见过最多的就是清除浮动还有图片icon

::selection用来改变浏览网页选中文本的默认效果，“::selection”来改变在浏览器中选中文本后的背景色与前景色。::selection在IE家族中，只有IE9+版本支持，在Firefox中需要加上其前缀“-moz”，:“::selection”只能设置两个属性，一个就是background,另一个就是color属性，设置其他属性是没有任何效果的。

html5语义话标签



这里写一下html5一些常用和新的语义话标签。

短语句

mark

用作标记文本，将标记的文本显示高亮，常用于搜索高亮。

例：< mark >标记文本< /mark >

标记文本

meter

定义度量衡：表示在一定范围内的值，如投票占比、使用量等。

属性：

value 实际度量的值，默认为0，可以为整数或者浮点数。

high 范围的上限值

low 范围的下限值

max 最大值，默认值是1

min 最小值，默认值是0

optimum 最佳的值，必须在min属性值与max属性值之间

progress

用于定义进度

属性：

value 实际进度的值，默认为0，可以为整数或者浮点数

max 最大值，默认值是1

min 最小值，默认值是0

time

定义时间和日期

< p >We open at < time >10:00 every morning.< /p >

< p >I have a date on < time datetime=“2008-02-14” >Valentines day< /time >.< /p >

datetime=“YYYY-MM-DDThh:mm:ssTZD”

ruby

在东亚使用，显示的是东亚字符的发音。

与 < ruby > 以及 < rt > 标签一同使用：

ruby 元素由一个或多个字符（需要一个解释/发音）和一个提供该信息的 rt 元素组成，还包括可选的 rp 元素，定义当浏览器不支持 “ruby” 元素时显示的内容。

< ruby >

漢 < rp >(< /rp >< rt >Kan< /rt >< rp >)< /rp >
字 < rp >(< /rp >< rt >ji< /rt >< rp >)< /rp >

< /ruby >


漢 (Kan)
字 (ji)

wbr

定义换行(Word Break Opportunity)

规定在文本中的何处适合添加换行符。

abbr

定义缩写

通过对缩写进行标记，能够为浏览器、拼写检查和搜索引擎提供有用的信息。

The < abbr title=“World Health Organization” >WHO< /abbr > was founded in 1948.

The WHO was founded in 1948.

bdi

定义隔离

允许设置一段文本，使其脱离其父元素的文本方向设置。

cite,blockquote,q

cite,blockquote,q都是引入的功能

cite标签定义作品（比如书籍、歌曲、电影、电视节目、绘画、雕塑等等）的标题。默认字体斜体。

blockquote标签之间的文本都会从常规文本中分离出来，经常会在左、右两边进行缩进，而且有时会使用斜体。

q标签定义一个短的引用。浏览器经常会在这种引用的周围插入引号。

output

< output > 标签定义不同类型的输出，比如脚本的输出。

code

格式化代码

< code >标签表示元素内包含的文本是使用某种程序语言编写的源代码。通常以等宽字体显示。

pre

定义预格式化的文本。被包围在 pre 元素中的文本通常会保留空格和换行符。而文本也会呈现为等宽字体。

object

定义一个嵌入的对象。可以使用此元素向 XHTML 页面添加多媒体。此元素运行规定插入 HTML 文档中的对象的数据和参数，以及可用来显示和操作数据的代码。

param

param 元素可以为插入 XHTML 文档的对象规定 run-time 设置，也就是说，此标签可为包含它的

新增标签

header,footer,section,aside,nav,article,hgroup

html5新增的用来描述特定区域结构的标签。

header：
定义文档的页眉，一般用来对主页的介绍。

footer：
定义文档或节的页脚，通常包含文档的作者、版权信息、使用条款链接、联系信息等等。

section：
定义文档中的节（section、区段）。比如章节、页眉、页脚或文档中的其他部分。

aside：
定义 article 以外的内容，一般用作文章的侧栏。

nav：
定义导航链接的部分。

article：
规定独立的自包含内容，独立于文档的其余部分。

hgroup：
用于对网页或区段（section）的标题进行组合。

details

定义元素的细节。

details元素用来描述文档或文档片段的信息，summary元素与details元素一同使用，用于说明文档的标题，同时，summary元素应为details元素的第一个子元素。通过一个简单的示例展示summary与details的使用。用summary定义细节标题。属性 open=“open”,默认下拉开启的状态。


i am details
这里是。。。

dialog

定义对话框或窗口,有open属性。

figure

figure标签规定独立的流内容（图像、图表、照片、代码等等），一般用作文档中插图的图像。

figure元素的内容应该与主内容相关，但如果被删除，则不应对文档流产生影响。

menu

标签定义菜单列表。当希望列出表单控件时使用该标签。

main

main标签规定文档的主要内容。

< main > 元素中的内容对于文档来说应当是唯一的。它不应包含在文档中重复出现的内容，比如侧栏、导航栏、版权信息、站点标志或搜索表单。

注意：在一个文档中，不能出现一个以上的 main 元素。main 元素不能是以下元素的后代：article、aside、footer、header 或 nav。

map

用于客户端图像映射。图像映射指带有可点击区域的一幅图像。有usemap属性和area标签。

address

定义文档作者/所有者的联系信息。

如果 address元素位于 body 元素内部，则它表示该文档作者/所有者的联系信息。

如果address 元素位于 article 元素内部，则它表示该文章作者/所有者的联系信息。

不应该使用 address 标签来描述邮政地址，除非这些信息是联系信息的组成部分。

address 元素通常被包含在footer元素的其他信息中。





表单元素


input

input标签用于搜集用户信息。

根据不同的 type 属性值，输入字段拥有很多种形式。输入字段可以是文本字段、复选框、掩码后的文本控件、单选按钮、按钮等等，例：

tel：
用于输入电话号码。

search：
用于提示用户输入要搜索的文字。search 和 text 之间的区别主要在于样式上.

url：
用于输入单个网址。

email：
用于输入单个电子邮件地址。如果指定了 multiple 属性，就可以输入多个电子邮件地址，以逗号分隔。

datetime：
用于输入时区设置为 UTC 的日期和时间。

datetime-local：
用于输入不含时区的日期和时间。

date：
用于输入不含时区的日期。

month：
用于输入含年份和月份但不含时区的日期。

week：
用于输入含年份和周数但不含时区的日期。例如，此格式的日期 2011-W05 表示 2011 年的第 5 周。

time：
用于输入含小时、分钟、秒和秒的小数部分但不含时区的时间值。

number：
用户输入数字。

range：
用于输入数字，但与 number 的区别在于无需输入具体数字。在大部分支持该类型的浏览器中，范围控件的实施形式为滑块。

color：
用于通过颜色池控件选择颜色。

label

label 标签为 input 元素定义标注（标记）。

label 元素不会向用户呈现任何特殊效果。不过，它为鼠标用户改进了可用性。如果您在 label 元素内点击文本，就会触发此控件。就是说，当用户选择该标签时，浏览器就会自动将焦点转到和标签相关的表单控件上。

label 标签的 for 属性应当与相关元素的 id 属性相同,for 属性可把 label 绑定到另外一个元素。

datalist

标签定义选项列表。与 input 元素配合使用该元素，来定义 input 可能的值。

datalist 及其选项不会被显示出来，它仅仅是合法的输入值列表。

使用 input 元素的 list 属性来绑定 dataliste 的id名。

keygen

用于表单的密钥对生成器字段。

当提交表单时，私钥存储在本地，公钥发送到服务器。

ins

定义已经被插入文档中的文本。

与 del 一同使用，来描述文档中的更新和修正。

samp

定义计算机程序的样本文本。

并不经常使用。只有在要从正常的上下文中将某些短字符序列提取出来，对它们加以强调的极少情况下，才使用这个标签。

fieldset

将表单内容的一部分打包，生成一组相关表单的字段。

当一组表单元素放到 fieldset 标签内时，浏览器会以特殊方式来显示它们，它们可能有特殊的边界、3D 效果，或者甚至可创建一个子表单来处理这些元素。

legend 为 fieldset 元素定义标题。

响应式布局 Meta标签

Meta标签

Meta提供关于HTML文档的元数据。元数据不会显示在页面上，但是对于机器是可读的。它可用于浏览器（如何显示内容或重新加载页面），搜索引擎（关键词），或其他 web 服务。

下列介绍一些常用的meta标签编码

1 .申明编码

< meta charset=’utf-8’ />

这个申明能让网页识别简体中文。

2 .优先使用 IE 最新版本和 Chrome

< meta http-equiv=”X-UA-Compatible” content=”IE=edge,chrome=1” />

3 .浏览器内核控制

< meta name=”renderer” content=”webkit|ie-comp|ie-stand”>

国内浏览器很多都是双内核（webkit和Trident），webkit内核高速浏览，IE内核兼容网页和旧版网站。而添加meta标签的网站可以控制浏览器选择何种内核渲染。

国内双核浏览器默认内核模式如下：

1). 搜狗高速浏览器、QQ浏览器：IE内核（兼容模式）

2). 360极速浏览器、遨游浏览器：Webkit内核（极速模式）

4 .页面关键字的设置

< meta name=”keywords” content=”your tags，” />

5 .页面描述

< meta name=“description”content=”120｜ 150 words” />

6 .搜索引擎索引方式

< meta name=“robots” content=”index,follow” />

all：文件将被检索，且页面上的链接可以被查询；

none：文件将不被检索，且页面上的链接不可以被查询；

index：文件将被检索；

follow：页面上的链接可以被查询；

noindex：文件将不被检索；

nofollow：页面上的链接不可以被查询。

7 .页面重定向和刷新

< meta http-equiv=”refresh” content=”0;url=” />

content内的数字代表时间（秒），既多少时间后刷新，如果加url,则会重定向到指定网页（搜索引擎能够自动检测，也很容易被引擎视作误导而受到惩罚）

8 .网页作者信息

< meta name=”author” content=”author name” />

hexo博客搭建

nodeJS的安装与测试

1.在官网下载最新的nodeJS文件，并安装。

2.在终端窗口对安装号的nodeJS进行测试，在桌面位置输入node-v并回车，得到v4.4.5的版本号，说明安装成功。

hexo的安装与测试

1.在桌面位置先创建一个blog的文件夹，在终端窗口中进入blog文件夹，再输入npm install hexo -g然后回车来安装hexo，若出现error，则是因为权限不够，这时输入sudo npm install hero -g 回车，然后会出现输入密码的命令，输入本机密码后便能成功安装。
2.终端窗口输入hexo init 进行对hexo目录的初始化，可以输入hexo new 文件名 来创建新blog。输入hexo server 启动本地服务，可以得到一个本地blog地址，浏览器进入即可看到当前的blog。若要退出此状态，输入Ctrl+C退出。

搭建github在线blog

创建github账号并与本地连接

1.在github官网创建一个自己的账号，记住用户名及邮箱，并通过邮箱认证。
2.在终端窗口中输入git，回车可得当前的git命令及常用用法。接下来进行对ssh-key的配置，输入ssh-keygen，并连续三次回车。再输入vim ~/.ssh/id_rsa.pub，回车得到ssh-key，复制ssh-key，进入自己的github设置，在ssh粘贴ssh-key。
3.接着配置github与本地机器验证，输入git config –global user.name”用户名”，回车输入git config - -global user.email “验证邮箱地址”。
4.在github的new repository新建博客域名仓库，输入用户名.github.io。

blog的底层设置

打开blog文件夹的_config.yml文件，可以对blog的一些基本情况进行设置，比如
title: 新的征程
subtitle: 新的开始
description: 记录学习点滴

最后设置最下面的deploy
type: git
repo: 域名仓库的https地址
其中域名仓库的https地址在自己的github博客的域名仓库中获得。

发布博客上传到线上

1.终端窗口中输入npm install hexo-deployer-git –save,回车对hexo进行git的安装和保存。
2.对编辑好的博客进行hexo server本地测试后，再输入hexo g，回车生成浏览器能识别的html和css文件，再输入 hexo d，回车将本地数据上传到自己的github博客中去。

这样就可以完成自己的博客了。^_^
